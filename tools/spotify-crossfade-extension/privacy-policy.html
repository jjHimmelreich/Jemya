Privacy Policy

Extension: Jam-ya Spotify Auto Fade  ·  Last updated: March 25, 2026

1. Overview

Jam-ya Spotify Auto Fade ("the Extension") is a Chrome browser extension that provides smooth crossfade volume control for the Spotify Web Player. This Privacy Policy explains what data the Extension accesses, how it is used, and what is never collected.

2. Data the Extension Accesses

2.1 Spotify OAuth Tokens

To control Spotify playback on your behalf, the Extension requires a Spotify OAuth access token. Depending on the connection mode you choose:

  • Jam-ya mode: The Extension reads a Spotify access token that the Jam-ya web app stores in your browser's localStorage on jam-ya.com. The token is forwarded to the Extension's background service worker and stored locally in chrome.storage.local on your device only. Token refresh calls are made directly to jam-ya.com/auth/refresh.
  • Custom App mode: You supply your own Spotify Client ID and Client Secret. These credentials are stored locally in chrome.storage.local on your device only. The OAuth flow uses chrome.identity.launchWebAuthFlow and the resulting token is stored locally on your device only.

2.2 Spotify Playback Data

The Extension polls the Spotify Web API (api.spotify.com) every few seconds to read your current playback state (track name, artist, progress, volume, play/pause status). This data is used solely to display the now-playing information in the popup and to determine when to apply crossfade. It is never stored beyond your current session or transmitted to any third party.

3. Data We Do NOT Collect

  • We do not collect any personal information.
  • We do not track browsing history or activity on any website.
  • We do not send any data to external analytics services.
  • We do not store any data on a remote server operated by this Extension.
  • We do not sell, share, or monetise any data in any form.
  • We do not read the content of any web page other than the specific localStorage key (jemya_token) on jam-ya.com.

4. Local Storage

The following data is stored locally on your device using chrome.storage.local and never leaves your browser:

  • Spotify access token and its expiry time
  • Spotify Client ID and Client Secret (Custom App mode only, if provided by you)
  • Extension settings: crossfade duration, fade curves, minimum volume, enabled state
  • Connection mode preference (Jam-ya or Custom App)

This data is retained until you log out within the Extension or uninstall the Extension.

5. Permissions Justification

  • storage — Save settings and tokens locally on your device.
  • alarms — Schedule periodic polling of playback state.
  • tabs — Detect when the Spotify Web Player tab is active to keep the service worker alive.
  • identity — Perform the OAuth flow via chrome.identity.launchWebAuthFlow in Custom App mode.
  • https://api.spotify.com/* — Call the Spotify Web API to read playback state and control volume/playback.
  • https://accounts.spotify.com/* — Exchange an OAuth authorisation code for an access token (Custom App mode).
  • https://jam-ya.com/* — Read the Spotify token from localStorage on jam-ya.com and refresh it when needed.
  • Content script on open.spotify.com — Maintain a keepalive connection to the background service worker while the Spotify Web Player tab is open.

6. Third-Party Services

The Extension communicates only with the following third-party services, strictly as required for its core functionality:

  • Spotify — playback control and OAuth
  • Jam-ya — token relay and refresh (Jam-ya mode only)

No other third-party services, CDNs, or analytics providers are used.

7. Children's Privacy

This Extension is not directed at children under the age of 13. We do not knowingly collect data from children.
